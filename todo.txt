0.1 release checklist
=====================

Required before publishing
--------------------------
- Add project metadata in pyproject.toml:
  - project URLs
- Clean generated build artifacts before committing:
  - build/
  - dist/
  - src/*.egg-info/
- Commit the current implementation changes.
- Run final validation:
  - uv run pre-commit run --all-files
  - uv run pytest
  - uv run pydoclint src
  - uv build

Recommended before publishing
-----------------------------
- Add a short changelog or release notes section for 0.1.0.
- Add README examples for:
  - write_single_register
  - write_multiple_registers
  - coils/discrete inputs
  - simulator background tasks
- Decide whether source distributions should include tests. If not, add
  packaging configuration to exclude tests from the sdist.
- Audit public exports in src/clear_modbus/__init__.py and ensure every intended
  public type is exported.
- Make exception behavior consistent in high-level client helpers. Some
  response-type and echoed-value mismatches still raise bare ValueError.
- Add a clean-environment wheel smoke test that installs the built wheel and
  imports clear_modbus.

Nice to have after 0.1
---------------------
- Add Modbus ASCII support.
- Add UDP or TLS transports if needed.
- Add a custom function-code registry.
- Add richer documentation under doc/ if the Sphinx setup is kept.
- Add CI to run pre-commit, tests, and build on pull requests.
